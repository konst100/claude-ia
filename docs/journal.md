## День 1 — 2026-05-01 (Phase 1: Foundation)

### Стартовая позиция

- Опыт разработки: только начинаю, AI-разработчиком хочу стать за 3–4 месяца.
- Установлен Claude Pro, в Cowork есть сессия-наставник.
- Папка пустая: `C:\claude-projects\claude_ia`.
- Ноль кода, ноль настройки, ноль решений по стеку.

### Что сделали

1. **Установили и авторизовали Claude Code (CLI).** Запускается через команду `claude` в терминале.
2. **Разобрались, что вообще такое Claude Code** — это отдельный инструмент Anthropic, не Desktop app и не Claude.ai. Он — мой младший партнёр-разработчик.
3. **Изучили два режима работы:** терминал (партнёрский диалог) и расширение в VS Code (быстрое исполнение).
4. **Выбрали стек (через диалог с Claude Code):**
   - Next.js 15 + App Router + TypeScript
   - Tailwind CSS v4 + shadcn/ui (Button, Card, Input)
   - ESLint
   - Планируется на следующих фазах: Supabase (DB + auth + storage), Clerk (auth), Vercel AI SDK + Anthropic SDK + OpenAI SDK (multi-provider), pgvector (RAG), Vercel (deploy).
5. **Сгенерировали стартовую страницу** — лендинг с шапкой, hero-блоком, четырьмя фича-карточками, статистикой, формой подписки и футером. Текст на русском.
6. **Запустили dev-сервер** (`npm run dev`) и убедились, что страница рендерится на `localhost:3000`.
7. **Инициализировали git, сделали 2 коммита:**
   - `862deeb` — Initial commit from Create Next App
   - `bd8c5c3` — feat: add shadcn/ui, landing page, and project README

### Что Claude Code сгенерировал сам (без явной просьбы)

- `CLAUDE.md` — но в нём только одна строка `@AGENTS.md` (это конвенция-указатель, нормально).
- `AGENTS.md` — но там только дефолтная техническая ремарка про Next.js 15, не настоящий контекст проекта.

**Урок:** автогенерация ≠ хорошее содержимое. Эти файлы нужно переписать самим. Это задача на день 3.

### Решения по процессу

- **«Vibe-coding ОК, но с пониманием».** После каждого шага Claude Code прошу объяснить, что и почему сделал.
- **Метафора команды:** я — директор и студент, Cowork — менеджер и наставник, Claude Code (терминал) — старший разработчик (партнёр), Claude Code (VS Code) — мидл-разработчик (исполнитель).
- **Правило одной активной сессии Claude Code на папку.** Иначе две сессии переписывают одни и те же файлы.

### Что узнал нового

- В терминале Claude Code приглашение `>` (а не `$`).
- Опции при запросе разрешения: `1. Yes` / `2. Yes, allow … during this session` / `3. No`.
- VS Code и Claude Code видят одни и те же файлы на диске, но не «общаются» через интерфейс — общая правда только в файлах.

### Итог дня

Рабочий сайт-визитка проекта на `localhost:3000`. Phase 1 (Foundation) — почти закрыта (осталась связка с GitHub и Vercel).

## День 2 — 2026-05-03 (организация знаний)

### Что сделали

1. **Зафиксировали в долгой памяти Cowork** ключевой контекст: проект, стек, метафора команды, моя роль как ученика, главный преподаватель — Cowork.
2. **Создали учебную папку курса** `C:\claude_academy\inaction1\Claude Code in Action\` с подпапками `notes/`, `prompts/`, `cheatsheets/` и файлами `README.md`, `00-progress.md`. Это **отдельная папка от проекта** — туда идут только конспекты и шпаргалки по курсу «Claude Code in Action», а сам код — в `claude_ia`.
3. **Перевели Cowork в режим «прямого доступа»** к папке `C:\claude-projects` — теперь Cowork (то есть наставник) может сам читать файлы проекта, не дёргая Claude Code в терминале на каждое чтение. Это сильно ускорит обучение.
4. **Начали миграцию документов из временной outputs-папки** прошлой Cowork-сессии в проект (`docs/`). Claude Code в терминале успел прочитать оба файла, но **упёрся в rate limit** Claude Pro и встал на паузу до сброса лимита.

### Проблема дня

- Claude Code в терминале дважды «домыслил» путь — пошёл искать файлы на Desktop и в Temp, хотя я давал точный путь. Прервал командой Ctrl+C, дал жёсткий повторный промпт со словами «никаких альтернатив, не угадывать». **Урок:** ИИ-агенты иногда «импровизируют». Первое слово в промпте должно быть конкретным действием, а не описанием задачи.
- Лимит Claude Pro срабатывает после крупных операций (чтение длинных файлов, копирование). Это ограничение плана, не баг.

### Что узнал нового

- `request_cowork_directory` в Cowork — это инструмент, через который наставник получает прямой доступ к папке. До этого работал только через посредничество Claude Code.
- Test-Path в PowerShell возвращает True/False — удобно для проверки путей.
